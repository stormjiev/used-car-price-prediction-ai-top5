## 术语说明

### OOF (Out-Of-Fold)

OOF是交叉验证中的核心概念，指**每个样本由"没见过它"的模型预测**。

**示例 (3折交叉验证)**:

```
假设6个样本: [A, B, C, D, E, F]，真实价格: [1000, 2000, 1500, 3000, 2500, 1800]

第1折: 训练[C,D,E,F] → 模型1预测[A,B] → A=1050, B=1900
第2折: 训练[A,B,E,F] → 模型2预测[C,D] → C=1600, D=2800
第3折: 训练[A,B,C,D] → 模型3预测[E,F] → E=2400, F=1750

OOF预测 = [1050, 1900, 1600, 2800, 2400, 1750] (合并所有折)
OOF MAE = (|1000-1050| + |2000-1900| + ... ) / 6 = 100
```

**为什么重要**:

1. **评估真实性能** - OOF MAE是对线上表现的可靠估计
2. **检测数据泄露** - OOF MAE异常低(如101)说明有问题
3. **用于Stacking** - OOF预测作为下一层模型的输入特征

### v2：建立可靠高分基线

**Plan摘要**：[mcode-v2/PLAN.md](mcode-v2/PLAN.md)

**问题背景**

- 前身 `mcode-v13` 自行重写高分方案，但特征、模型训练和最终执行链存在偏差。
- 旧版最终没有真正完成NN融合，线上最好约为 `482.7833`。
- v2选择直接对照参考高分方案代码，修正数据路径和文件名后完整执行。

**核心实现**

- 树模型使用精选约83个特征，NN使用约29个归一化特征。
- LightGBM和CatBoost使用10-Fold；NN使用5-Fold。
- 树模型在`log1p(price)`空间训练，NN直接预测原始价格。
- LGB和CAB先进行Bayesian Ridge Tree Stacking，再与NN进行50/50融合。

**实际结果**

| 组件 | 结果 |
|------|------|
| LightGBM OOF | 446.61 |
| CatBoost OOF | 436.48 |
| NN OOF | 436.01 |
| Tree Stack线上 | 423.5920 |
| NN线上 | 415.3266 |
| 最终Tree+NN融合线上 | **405.1573** |

**关键结论**

1. 真正的跃升来自完整执行异质Tree+NN双路方案，而不是局部调参。
2. Tree模型与NN使用不同特征和目标空间，形成了可被融合利用的误差互补。
3. 简单而稳定的NN特征已经足以显著改善树模型。
4. v2提供了后续版本可以继续扩展的可靠基线。

**传递给v3的资产**

- 已验证的Tree/NN双路框架。
- 约83个Tree特征与29个NN特征。
- Tree Stack、NN预测和最终融合方法。
- `405.1573`线上基线。

## 核心教训总结 (2025-12-14更新)

### 1. 特征工程定律
```
特征数量 ≠ 特征质量
573特征 (OOF 450) < 93特征 (OOF 446) < 55特征 (OOF 406)

关键: 高分方案的83特征是精选的，不是越多越好

- 同时保留造成多重共线性
- v_0-v_14不应与交叉特征共存
```

### 2. 融合定律 ⭐ 极其重要 (更新)
```
If |MAE_A - MAE_B| > 50:
    ❌❌ 绝对禁止融合！必然失败 (v27: LGB 505 + CatBoost 457 → 470)

If |MAE_A - MAE_B| > 10:
    ❌ 禁止融合！几乎必然失败

If |MAE_A - MAE_B| < 5:
    ⚠️ 可以尝试融合
    需先验证OOF融合效果

同质融合优于异质融合:
    ✅ 3x CatBoost (454, 455, 454) → 446 (改进8点)
    ❌ LGB (505) + CatBoost (457) → 470 (恶化)
```

### 3. 模型选择定律 (新增)
```
相同特征下的模型性能对比:
    CatBoost: OOF ~454 (推荐!)
    LightGBM: OOF ~505 (差距50点)

原因分析:
    - CatBoost对类别特征处理更好
    - CatBoost对缺失值处理更鲁棒
    - CatBoost正则化机制不同

结论: 优先使用CatBoost作为基础模型
```

### 失败原因分析 ⭐⭐

#### 定律11: OOF过拟合陷阱

```
OOF改进: +2830点 (3458 → 628)
线上结果: -1.29点 (401.39 → 402.68)

过拟合比例: 2830 / 1.29 = 2194倍！

根本原因: 使用y_pred作为残差模型特征
- OOF中: y_pred与残差有强相关性(r=-0.4852)
- 测试集: y_pred是新数据的预测，相关性不成立
- 结果: 残差模型学到的是"如何在训练集上修正"
        而不是"如何泛化到测试集"
```

#### 信息泄漏机制

```python
# 训练时
residual = y_true - y_pred_oof  # y_pred_oof是训练集的OOF预测
X_features = [原始特征, y_pred_oof]  # ❌ y_pred_oof包含训练集信息

# 残差模型学到:
# "当y_pred_oof高时，残差倾向于负（高估）"
# "当y_pred_oof低时，残差倾向于正（低估）"
# 这在OOF中完美成立！

# 测试时
y_pred_test = model.predict(X_test)  # 测试集预测
# 但测试集的y_pred_test与残差的关系不同于训练集
# 因为测试集没有"真实值"来计算残差
# 残差模型的修正方向可能完全错误！
```

### 量化分析

| 阶段 | 预期 | 实际 | 差距 |
|------|------|------|------|
| Phase 1: 误差分析 | 发现可修正偏差 | r=-0.4852 | ✅ |
| Phase 2: 残差训练 | R²>0.1 | R²=0.9675 | ⚠️ 过高 |
| Phase 3: OOF验证 | 改进<500 | 改进2830 | ❌ 过大 |
| Phase 5: 线上验证 | 改进0.5-1 | **退步1.29** | ❌❌ |

### 核心教训

#### 定律12: 特征泄漏检测

```
当OOF改进率 > 50%时，高度怀疑信息泄漏

v32情况:
  OOF改进率 = 2830/3458 = 82% ❌❌ 严重泄漏

检测方法:
  1. 残差预测R² > 0.5 → 可疑
  2. OOF改进 > 原始MAE的20% → 可疑
  3. 使用y_pred作为特征 → 高风险
```

#### 定律13: 残差学习的正确做法

```
❌ 错误做法 (v32):
  - 使用y_pred作为残差模型特征
  - 在全量数据上训练残差模型
  - 期望OOF改进能泛化到测试集

# Mean Encoding特征（参考422方案）
MEAN_ENCODE_FEATURES = ['model', 'brand', 'v_0', 'v_3', 'v_8', 'v_12']

# 平滑参数
ALPHA = 5.0

# K-fold OOF编码（避免数据泄漏）
for feat in MEAN_ENCODE_FEATURES:
    for fold in range(5):
        # 用其他fold统计编码当前fold
        train_stats = train_data[other_folds].groupby(feat)['price'].agg(['sum', 'count'])

## v44 经验总结 (2025-12-20)

### 结果

- Baseline OOF MAE: 470.59 (单LGB)
- Pseudo OOF MAE: 464.60 (扩展训练集)
- 改进: +6.0
- 止损触发(>410)，方案无效

### 核心方法

Pseudo Labeling：使用11个高质量模型的测试集预测生成伪标签

- 模型：CatBoost(3) + LightGBM(2) + NN(5) + Tree Stack(1)
- 置信度筛选：CV < 0.05 → 44,363样本(88.7%)
- 扩展训练集：150K → 194K

### 失败原因

1. **基线模型过弱**: 单LGB OOF 470 vs v4完整pipeline 401
   - 未使用CatBoost + NN stacking
   - 未使用完整特征工程pipeline
2. **伪标签来源不独立**: 伪标签由同一套特征的模型预测
   - 模型"记住"自己的预测模式
   - 无法带来真正的信息增益
3. **评估方式有偏**: 扩展数据集CV包含伪标签样本
   - Fold内MAE低(376-398)是假象
   - 实际OOF仍为464

### 定律更新

```
定律37: Pseudo Labeling需基于强模型(OOF<410)，弱基线扩展无效
定律38: 伪标签应来自多样化模型(不同特征体系)，同源模型无信息增益
定律39: 评估时必须排除伪标签样本，避免"自我验证"偏差
```

### 结论

**v34 (401.36) 仍为方法论天花板**

## v51 经验总结 (2025-12-22) ❌❌ 对抗验证+TabNet双失败

### 目标
通过对抗验证减少OOF到线上的8点Gap，突破MAE 400

### 策略A: 对抗验证 + 样本加权

**原理**:
- OOF MAE ~409 vs 线上 MAE ~401 存在 ~8点系统差距
- 假设：训练集和测试集分布存在差异
- 方法：训练分类器区分train/test，给"更像测试集"的训练样本更高权重

**执行结果**:

| 指标 | 数值 | 结论 |
|------|------|------|
| 对抗验证 AUC | **0.4988** | ≈0.5 (随机猜测) |

| 无权重LGB OOF | 485.37 | - |
| 差异 | +0.10 | ❌ 无改进 |

**失败原因**:
```
AUC = 0.4988 ≈ 0.5 说明：
1. 训练集和测试集分布几乎完全相同
2. 分类器无法区分样本来源
3. OOF到线上的8点Gap不是分布差异造成的

真正原因可能是：
- 随机性/方差
- 过拟合训练集
- 后处理在测试集上更有效
```

### 策略B: TabNet深度学习

**原理**:
- TabNet是专门为表格数据设计的深度学习架构
- 可能带来与树模型不同的预测多样性

**执行结果**:

**失败原因**:
```
1. OOF 553 远差于树模型融合的 408
2. 相关性 0.9964 → 几乎没有多样性
3. TabNet在此数据集上表现不如传统树模型

可能原因：
- 数据量不够大 (15万样本)
- 特征已高度工程化，深度学习无优势
- 超参数未充分调优
```

### 定律更新

```
定律51: 对抗验证前提是分布差异存在
  - AUC > 0.6 才说明有分布差异
  - AUC ≈ 0.5 说明分布相似，方法无效
  - 本数据集 AUC=0.4988，对抗验证不适用

  - v13系列 OOF ~409 vs 线上 ~401
  - 但对抗验证显示分布相似
  - Gap可能来自：随机性、后处理效果、过拟合

定律53: TabNet不是万能解
  - 在高度工程化特征上，树模型仍占优
  - 深度学习需要更多数据或原始特征
  - 高相关性(0.9964)意味着融合无增益

定律54: 混入低质量预测必然退步
  - v50(OOF 414) 混入10% → 线上+1.34
  - 质量差距越大，混入比例应越小
